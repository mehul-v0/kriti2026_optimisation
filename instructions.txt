# Velora Mobitech — Frontend & UX Ideation Report (Revised Edition)

## Executive Vision

The goal is to create a **data-rich yet visually digestible** interface that transforms complex route optimization into an intuitive, compelling narrative. Every screen should answer: *"How much are we saving, and why should I trust this system?"*

The interface prioritizes **transparency** — users see exactly what constraints were honored, which were violated, and why the system made specific decisions.

---

## Information Architecture

### Page Structure Overview

The application follows a **three-act story structure**: Setup (upload & understand data) → Transformation (optimization) → Resolution (results, validation, and insights).

**Primary Pages:**
1. Landing / Dashboard
2. Data Upload & Configuration
3. Data Insights Preview
4. Optimization Processing View
5. Results Overview
6. Constraint Validation Report
7. Route Explorer (Interactive Map)
8. Vehicle Fleet Analysis
9. Employee Assignment Details
10. Cost & Savings Breakdown
11. Export & Reports

### 7. Route Explorer (Interactive Map)

**Purpose:** The visual centerpiece — see all routes geographically

**Layout Philosophy:**
Full-screen immersive map experience with slide-out panels for details. The map is the hero; controls are minimal and contextual.

**Map Canvas:**
Full-viewport map using a clean, professional base style (light theme recommended for clarity). The map shows the actual geographic region covered by the routes.

*[Image suggestion: The map itself serves as the visual — ensure a clean, modern map tile style like Mapbox Light or similar]*

---

**Route Rendering:**

Each vehicle's route is rendered as a distinct colored polyline following actual road paths (not straight lines). Colors are assigned from a carefully chosen palette ensuring adjacent routes are visually distinguishable.

**Pickup points:** Circular markers with employee initials or number
**Drop-off points (office):** Diamond or building icon marker, distinctly styled
**Vehicle starting point:** Small vehicle icon in the route's color

Routes show directional indicators (small arrows along the path) to indicate travel direction.

**Critical: Office Location Handling**

When a vehicle completes a trip (drops employees at office) and then starts another trip, the office location must be clearly marked and labeled on the route. The office appears as a prominent landmark marker with a building icon and label "Office — Destination"

Between trips, a visual break or dotted line indicates the vehicle's return or repositioning.

---

A collapsible left sidebar (default: collapsed to show just vehicle icons vertically).

Expanded view lists all vehicles with:
- Vehicle ID

- Trip count for this vehicle (e.g., "2 trips")
- Total passengers
- Total distance

**Interaction:** Clicking a vehicle:
- Highlights only that vehicle's route(s)
- Dims all other routes to 20% opacity
- Zooms map to fit that vehicle's route bounds
- Opens detail panel on right

**"Show All" button** resets to display all routes.

**Section C: Cost Methodology Explanation**

A clear text block explaining the calculation:

**"How We Calculate Costs"**

**Baseline Cost:** For each employee, the baseline cost is provided directly in the input data. This represents the estimated cost of an individual ride for that employee. The total baseline cost is the sum of all individual baseline costs.

**Optimized Cost:** The optimized cost is calculated as the sum of operational costs for all vehicles used:
- For each vehicle: Distance traveled (km) × Cost per kilometer (from vehicle data)
- Total = Sum across all vehicles and all trips

**Savings:** The difference between baseline and optimized costs.

## Design System Principles

## Image and Illustration Guidance

Throughout the application, use imagery strategically to enhance understanding and create visual appeal without overwhelming data-heavy pages.

### Landing Page Hero
*Abstract network visualization with animated nodes and connections. Suggests intelligent optimization without being literal. Brand colors (navy, teal). Subtle motion.*

### Empty States
*When no data exists (e.g., first visit, no history), show friendly illustrations of the feature being explained. Example: An illustration of a route being optimized with a progress indicator for the "no session history yet" state.*

### Processing Page Background
*Very subtle, abstract route-like lines drifting slowly at low opacity. Creates atmosphere without distraction.*

### Map Base Style
*Clean, modern map tiles. Light background with minimal labels. Roads clearly visible. No satellite imagery — vector only.*

### Vehicle Icons
*Consistent set of vehicle silhouettes: motorcycle, sedan, minivan. Used throughout in cards, map markers, and tables.*

### Status Illustrations
*For the constraint compliance page, consider a small illustration header showing a checklist or validation theme. For cost savings, a subtle chart/growth visual.*

### Export Page
*Small icons for each export type (Excel icon, PDF icon, JSON icon) — recognizable file type representations.*

## Interaction Patterns

### Progressive Disclosure

Never overwhelm users with all information at once. Every page follows: Summary first → Details on demand.

Tables have expandable rows. Cards have detail drawers. Sections are collapsible. Users control how deep they go.

### Contextual Help

Small "?" or "ⓘ" icons next to complex metrics or unfamiliar terms. Clicking/hovering reveals a tooltip or popover explaining the concept.

Example: Next to "Baseline Cost" — "The individual ride cost for this employee as specified in your input data"

### Persistence with Local Storage

Session history and settings persist via browser local storage. On return visits, users see their previous sessions and can pick up where they left off.

Clear data option always available for privacy.

### Responsive Behavior

**Desktop (1200px+):** Full experience as described, multi-column layouts, side-by-side panels.

**Tablet (768-1199px):** Collapsed sidebars by default, stack cards vertically, full-width tables with horizontal scroll.

**Mobile (< 768px):** Map becomes primary view on route explorer. Tables become scrollable cards. Bottom sheet patterns for details. Simplified navigation.

### Keyboard Navigation

All interactive elements accessible via keyboard. Logical tab order. Visible focus states. Screen reader labels for icons and graphics.

### Error Handling

Clear, friendly error messages. Never just "Error occurred" — always explain what went wrong and how to fix it.

Validation errors highlight specific fields/rows. Success confirmations are brief and dismissible.
